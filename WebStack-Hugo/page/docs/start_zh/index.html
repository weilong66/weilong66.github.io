🚀开始使用

本地安装部署

1、安装配置

  1. 克隆项目到本地
  2. 修改 config.toml 配置项,按照你自己的需求修改。
  3. 在 webstack.yml 中添加新网站，修改和删除已有网站。
  4. 在项目根目录下执行 hugo，执行后等待hugo生成静态文件，静态文件存放在根目录下 public 文件夹中。
    • 注意，项目附带的hugo为Window版本，其它系统请自行下载替换成你的系统支持的Hugo版本。
  5. 可以执行下面命令运行一个开发服务器，用于实时预览网站效果。（不建议使用该服务器部署网站） 
    hugo server

2、本地部署

  1. 网站效果满意后，执行 hugo.exe 生成静态文件。
  2. 将在根目录下 public 文件夹中生成的静态文件托管到Nginx或其它Web服务器上即可。

使用github.io和GitHubActions自动部署

  1. fork 或 clone 此项目到你的GitHub仓库。
  2. 从你的GitHub仓库中克隆项目到本地
  3. 打开 .github/workflows/HugoAction.yml ，找到下面这些配置项并修改（未提及的保持默认即可） 
    - name: Deploy Pages
      uses: peaceiris/actions-gh-pages@v3
      with:
        # 具体发布到哪个仓库，需要把这里的"yourName"改成你自己的GitHub用户名
        external_repository: yourName/yourName.github.io
  4. 按照步骤创建操作token：
    1. 点击：GitHub 个人账号 -> Settings -> Developer settings -> Personal access tokens -> Tokens (classic) -> New personal access token (classic)
    2. 点击：Generate new token -> Generate new token (classic) 生成新令牌，
      • Note：输入任意名称
      • Expiration：选择生命周期。推荐No expiration或者尽量长一些。
      • Select scopes：勾选repo:status、repo_deployment、public_repo
      • 最后点击 Generate token 生成令牌，并复制保存。
  5. 设置Token
    1. 从你的仓库中打开此项目
    2. 依次进入：Settings -> Security -> Secrets and variables -> Actions
    3. 点击 New repository secret
    4. Name 使用PERSONAL_ACCESS_TOKEN，Secret 使用刚生成的Token
  6. Github中创建一个新项目，名称设置为 <你的用户名>.github.io
  7. 配置完成后，每次本地推送项目到远程仓库的main分支时，就会自动通过GitHubActions下载运行hugo生成静态资源，然后将静态资源推送到<你的用户名>.github.io远程仓库中。

🛠️配置说明

config.toml

主要的配置信息都集成到了 config.toml，一键完成各种自定义的配置。
每个配置项都有注释，这里不再解释。

建议修改的配置项：

  • baseURL：基础URL，需要设置成你的域名根路径，即访问主页的路径；如果为空，则使用使用相对路径。
    • 例如：网站地址为 https://yourname.github.io/WebStack-Hugo/，则baseURL也需要设置成该值，否则网站中所有资源路径会出错。
    • 也可以使用hugo的启动命令参数 --baseURL= 来指定基础URL。
  • title：网站标题。
  • textPreLoad：网站加载动画显示的文本。
  • copyright：网站底部文本。

webstack.yml

导航的各个信息都集成在 data 文件夹中.yml后缀文件中，方便后续增删改动。

  • webstack.yml：所有网站导航信息。
  • headers.yml：友情链接中的网站信息。
  • friendlinks.yml：配置顶部“关于”菜单下的网站信息。

具体参数说明：

- taxonomy: 常用网站                        # 左侧导航条顶级菜单名称。
  icon: far fa-star                        # 左侧导航条顶级菜单最左侧图标。使用Font Awesome图标，直接输入图标对应名称即可。
  list:                                
    - term: 常用网站                        # 导航条子菜单名称。
      links:                           
        - title: 邮箱                       # 网站名称
          logo: 邮箱.png                    # 网站Logo
          url: http://mail.email.com/       # 网站URL
          description: 邮箱网站。           # 网站介绍

icon：左侧导航条顶级菜单最左侧图标。使用Font Awesome 6.7.2图标，直接输入图标对应名称即可加载。

logo: 网站logo。

  • 默认使用 config.toml 中的 baseURL+logosPath 配置值作为前缀，拼接 logo 的值作为网站LOGO图片的资源路径。
  • 如果使用 / 开头，则只使用 config.toml 中的 baseURL 配置值作为前缀，拼接 logo 的值作为完整URL设置为图片路径。
  • 如果使用 http:// 或 https:// 开头，则直接将 logo 的值作为完整URL设置为图片路径。
  • 如果不设置值，且 config.toml 中的 websiteLogo 值为 true，则会使用 url 的值拼接 favicon.ico 作为网站logo图片的资源路径。
    （因为大部分网站logo路径都是 https://<网站域名>/favicon.ico）
